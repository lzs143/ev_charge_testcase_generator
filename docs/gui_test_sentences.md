# GUI 自然语言测试语句集

本文档用于手工测试 `examples/gui_tool.py` 的语义抽取、事件语义归一化和可执行语义构建效果。建议同时切换“规则抽取”和“MacBERT-GlobalPointer 模型抽取”两种方式进行对比。

重点观察字段：

- 场景类型
- 工况类型
- 测试阶段
- 故障类型
- 报文类型
- 通信对象
- 事件语义：测试刺激、期望响应、检查点
- 参数
- 无关输入是否被拒绝

## 1. 直流正向测试

```text
发送CHM报文，查看BMS是否正确回复BHM报文
```

```text
发送CRM辨识报文后，等待BMS回复BRM车辆辨识报文
```

```text
充电参数配置阶段发送CML报文，检查车辆是否发送BCP报文
```

```text
发送CRO准备就绪报文，检查BMS是否回复BRO报文
```

## 2. 直流异常测试

```text
发送周期错误的CHM报文，检查BMS是否回复BEM错误报文
```

```text
发送报文ID错误的CRM报文，报文ID设置为0x123，检查BMS是否回复BEM报文
```

```text
将CRO报文的准备就绪状态置为非法值，检查BMS是否回复BEM错误报文
```

```text
充电阶段未收到BCL报文并等待超时后，检查充电机是否发送CEM错误报文
```

```text
发送DLC长度错误的CRO报文，检查BMS是否进入异常处理流程
```

## 3. 直流参数与边界测试

```text
车辆完成握手后，设置直流充电电压为500V，电流为30A
```

```text
将BCP报文最高允许充电电压设置为900V，检查系统是否进入异常处理流程
```

```text
将BCL报文需求电流设置为非法值，检查充电机是否记录故障信息
```

## 4. 交流正向测试

```text
交流连接确认阶段，设置CC阻值为680Ω，检查车辆是否完成连接确认
```

```text
交流充电过程中，检查CP信号占空比是否保持正常
```

## 5. 交流异常测试

```text
交流充电过程中，当CP信号异常时，系统应停止充电并提示异常状态
```

```text
交流连接确认阶段，将CC阻值设置为2000Ω，检查车辆是否识别为异常连接
```

```text
交流充电过程中，断开保护接地PE，检查系统是否停止供电并记录故障
```

## 6. 连接与状态类测试

```text
直流充电过程中，断开车辆接口连接，检查系统是否停止输出
```

```text
充电过程中BMS通信中断时，系统应停止输出并记录故障信息
```

```text
充电结束阶段收到BST报文后，检查充电机是否回复CST报文
```
