# 语义标签体系与标注规范

## 1. 适用范围

本文档定义新能源汽车充电测试需求语义抽取任务的标签体系，用于构建
MacBERT-GlobalPointer 多任务语义抽取模型的数据集。

标签体系同时覆盖两类标准测试需求：

- GB/T 34658-2025：非车载传导式充电机与 BMS/EVCC 之间的通信协议一致性测试。
- GB/T 34657.2-2017：电动汽车传导充电互操作性测试中车辆侧直流/交流充电相关测试。

模型输入为自然语言测试需求，输出包括：

- 实体抽取结果：报文、信号、对象、接口、动作、故障表达、参数等。
- 分类结果：标准来源、场景类型、系统类别、测试层级、测试阶段、故障类型、动作意图等。
- 可执行语义候选：供时序知识库、动作库和规则校验模块继续补全。

## 2. 实体标签

| 标签 | 含义 | 示例 |
| --- | --- | --- |
| `MESSAGE` | 协议报文名称 | `CHM`、`BEM`、`CML`、`BMT` |
| `SIGNAL` | 物理信号或控制信号 | `CP`、`CC`、`CC2`、`S+`、`S-` |
| `OBJECT` | 测试对象或通信对象 | `BMS`、`EVCC`、`SECC`、`车辆`、`充电机` |
| `INTERFACE` | 充电接口、触点或连接件 | `车辆插座`、`车辆插头`、`PE`、`DC+`、`DC-` |
| `COMPONENT` | 被控部件或电气部件 | `K5`、`K6`、`电子锁`、`辅助电源`、`接触器` |
| `TEST_LAYER` | 协议一致性测试层级 | `物理层`、`数据链路层`、`应用层` |
| `ACTION` | 测试动作或期望动作 | `发送`、`检查`、`等待`、`闭合`、`断开`、`停止` |
| `FAULT_EXPR` | 自然语言中的故障表达 | `错误周期`、`S+断路`、`PE断针`、`绝缘故障` |
| `PARAM_NAME` | 参数名称 | `周期`、`报文ID`、`电压`、`电流`、`占空比` |
| `PARAM_VALUE` | 参数取值 | `500ms`、`0x123`、`500V`、`30A`、`5%` |
| `FIELD_NAME` | 报文字段名或 SPN 名称 | `SPN2601`、`辨识结果`、`最高允许充电电压` |
| `EXPECTED_EXPR` | 预期响应或判据表达 | `回复BEM报文`、`停止充电`、`禁止行驶` |
| `TEST_CONDITION` | 前置条件或触发条件 | `车辆接口连接后`、`充电过程中`、`收到BRO后` |

## 3. 报文与对象覆盖

### 3.1 协议报文 `MESSAGE`

GB/T 34658-2025 协议一致性测试应覆盖 GB/T 27930 相关报文。

| 发送侧 | 报文 |
| --- | --- |
| 充电机/SECC | `CHM`、`CRM`、`CTS`、`CML`、`CRO`、`CCS`、`CST`、`CSD`、`CEM` |
| 车辆/BMS/EVCC | `BHM`、`BRM`、`BCP`、`BRO`、`BCL`、`BCS`、`BSM`、`BMT`、`BMV`、`BSP`、`BST`、`BSD`、`BEM` |

标注规则：

- `BMS`、`EVCC`、`SECC` 是 `OBJECT`，不是 `MESSAGE`。
- 同一句中出现多个报文时，均标注为 `MESSAGE`。
- 复合写法如 `BST/BSD` 应拆分为两个 `MESSAGE` 实体。

### 3.2 互操作性测试对象

GB/T 34657.2-2017 直流/交流互操作性测试除通信报文外，还应覆盖接口和电气对象。

| 类别 | 对象 |
| --- | --- |
| 通信对象 | `车辆`、`BMS`、`EVCC`、`充电机`、`SECC` |
| 接口触点 | `PE`、`DC+`、`DC-`、`S+`、`S-`、`CC1`、`CC2`、`CP`、`CC` |
| 控制部件 | `K5`、`K6`、`电子锁`、`接触器`、`车辆接口`、`车辆插座`、`车辆插头` |
| 电源/检测对象 | `低压辅助电源`、`绝缘检测回路`、`检测点1`、`检测点2`、`检测点3`、`R3` |

## 4. 分类标签

### 4.1 标准来源 `standard_source`

| 标签值 | 含义 |
| --- | --- |
| `GB/T 34658-2025` | 通信协议一致性测试 |
| `GB/T 34657.2-2017` | 车辆传导充电互操作性测试 |
| `GB/T 27930-2023` | 数字通信协议语义约束来源 |
| `GB/T 18487.1-2023` | 传导充电系统控制导引和安全约束来源 |

### 4.2 场景类型 `scene_type`

| 标签值 | 含义 |
| --- | --- |
| `AC` | 交流充电测试 |
| `DC` | 直流充电测试 |

### 4.3 系统类别 `system_class`

GB/T 34658-2025 包含 A 类、B 类、C 类系统测试例。语义标注中保留该标签，便于后续扩展协议版本和系统能力。

| 标签值 | 含义 |
| --- | --- |
| `A` | A 类系统 |
| `B` | B 类系统 |
| `C` | C 类系统 |
| `unknown` | 文本中未指定系统类别 |

### 4.4 测试层级 `test_layer`

| 标签值 | 含义 |
| --- | --- |
| `physical_layer` | 物理层测试 |
| `data_link_layer` | 数据链路层测试 |
| `application_layer` | 应用层测试 |
| `interoperability` | 互操作性流程或接口测试 |

### 4.5 工况类型 `condition_type`

| 标签值 | 含义 |
| --- | --- |
| `normal` | 正常工况 |
| `fault` | 故障或异常工况 |

### 4.6 测试类型 `test_type`

| 标签值 | 含义 |
| --- | --- |
| `positive` | 正向一致性或互操作性测试 |
| `negative` | 故障注入、边界或异常测试 |

### 4.7 测试阶段 `test_stage`

| 标签值 | 典型报文/信号/对象 |
| --- | --- |
| `测试准备阶段` | 接口连接、低压辅助电源、CAN 配置 |
| `低压辅助上电及充电握手阶段` | `CHM`、`BHM`、`BEM` |
| `辨识阶段` | `CRM`、`BRM`、车辆辨识 |
| `时间同步阶段` | `CTS` |
| `充电参数配置阶段` | `CML`、`BCP` |
| `充电准备就绪阶段` | `CRO`、`BRO` |
| `充电阶段` | `BCL`、`BCS`、`BSM`、`BMT`、`BMV`、`BSP`、`CCS` |
| `充电结束或中止阶段` | `BST`、`CST`、`BSD`、`CSD` |
| `错误处理阶段` | `BEM`、`CEM` |
| `直流接口连接确认阶段` | `PE`、`DC+`、`DC-`、`S+`、`S-`、电子锁 |
| `直流控制导引阶段` | `K5`、`K6`、低压辅助电源、绝缘检测 |
| `交流连接确认阶段` | `CC`、`车辆接口`、`电子锁` |
| `交流控制导引阶段` | `CP`、`CC`、占空比、检测点电压 |
| `充电与行驶互锁阶段` | 车辆行驶互锁、接口锁止 |

## 5. 故障类型 `fault_type`

故障类型分为协议一致性故障和互操作性/接口故障两组。模型可输出标准化 `fault_type`，同时保留原文中的 `FAULT_EXPR`。

### 5.1 协议一致性故障

| 标签值 | 说明 |
| --- | --- |
| `报文内容错误` | 报文字段、数据域、枚举值、长度、校验等内容异常 |
| `报文周期错误` | 报文发送周期不符合要求 |
| `报文ID错误` | CAN ID 或报文标识异常 |
| `报文长度错误` | DLC 或报文长度不符合要求 |
| `报文格式错误` | 帧格式、编码格式或字段排列不符合要求 |
| `报文顺序错误` | 报文发送顺序不符合协议流程 |
| `报文超时` | 超过规定时间未收到指定报文 |
| `通信中断` | 通信链路中断、无响应或持续未通信 |
| `错误报文响应` | 期望 BEM/CEM 等错误报文响应 |
| `状态机跳转错误` | 被测对象未进入期望阶段或状态 |

### 5.2 直流互操作性与安全故障

| 标签值 | 说明 |
| --- | --- |
| `绝缘故障` | 绝缘检测失败或绝缘阻值异常 |
| `保护接地故障` | PE 断针、接地连续性异常 |
| `S+断路` | 直流通信线 S+ 断开 |
| `S-断路` | 直流通信线 S- 断开 |
| `S+S-短路` | S+ 与 S- 短路或通信线短接 |
| `K5状态异常` | 车辆侧 K5 接触器状态异常 |
| `K6状态异常` | 车辆侧 K6 接触器状态异常 |
| `低压辅助电源异常` | 辅助电源电压异常、断开或不满足上电条件 |
| `电子锁异常` | 锁止失败、解锁失败或锁止状态异常 |
| `车辆接口断开` | 车辆插头/插座断开、接口连接异常 |
| `充电与行驶互锁异常` | 充电连接状态下车辆仍允许行驶或互锁失败 |
| `车辆中止充电` | 车辆主动中止或无法继续充电 |
| `充电机中止充电` | 充电机主动中止或停止输出 |

### 5.3 交流互操作性与控制导引故障

| 标签值 | 说明 |
| --- | --- |
| `CP信号异常` | CP 电压、频率、占空比或状态异常 |
| `CC信号异常` | CC 电阻、电压或连接确认异常 |
| `CC2信号异常` | 直流接口 CC2 检测异常 |
| `CP断路` | CP 线路断开 |
| `CC断路` | CC 线路断开 |
| `PE断路` | 保护接地线路断开 |
| `控制导引边界异常` | 检测点电压、占空比或状态边界异常 |

### 5.4 电气参数与通用安全故障

| 标签值 | 说明 |
| --- | --- |
| `电压越界` | 电压值超过允许范围 |
| `电流越界` | 电流值超过允许范围 |
| `过压` | 电压高于上限 |
| `欠压` | 电压低于下限 |
| `过流` | 电流高于上限 |
| `温度异常` | 温度过高或温度采样异常 |
| `SOC异常` | SOC 值异常或超出合理范围 |
| `急停` | 急停或紧急停止触发 |

## 6. 动作意图 `action_intent`

动作意图为多标签分类，同一句需求可以同时包含多个意图。

| 标签值 | 说明 |
| --- | --- |
| `发送报文` | 测试系统主动发送某类报文 |
| `等待报文` | 等待被测对象发送报文 |
| `检查响应` | 检查报文、状态或信号是否满足预期 |
| `设置参数` | 设置电压、电流、周期、ID、字段值等参数 |
| `故障注入` | 注入报文、信号、接口或通信异常 |
| `断开连接` | 断开接口、触点、线束或通信链路 |
| `闭合连接` | 闭合开关、接触器或连接回路 |
| `模拟边界` | 设置控制导引、电压、电流等边界条件 |
| `停止输出` | 停止充电或停止功率输出 |
| `记录故障` | 记录或上报故障信息 |
| `恢复环境` | 故障恢复、复位或恢复正常连接 |

## 7. 参数名 `PARAM_NAME`

| 参数类别 | 参数名示例 |
| --- | --- |
| 报文参数 | `周期`、`报文ID`、`DLC`、`字段`、`SPN`、`枚举值` |
| 电气参数 | `电压`、`电流`、`绝缘电阻`、`温度`、`SOC` |
| 控制导引参数 | `占空比`、`频率`、`检测点电压`、`CC电阻` |
| 时间参数 | `超时时间`、`持续时间`、`等待时间`、`响应时间` |
| 接口参数 | `锁止状态`、`连接状态`、`接触器状态` |

## 8. 标注原则

### 8.1 实体边界

实体边界应尽量标注最小但完整的业务语义单元。

```text
发送错误周期CHM报文，检查BMS能否正确回复BEM报文
```

| 片段 | 标签 |
| --- | --- |
| `发送` | `ACTION` |
| `错误周期` | `FAULT_EXPR` |
| `CHM` | `MESSAGE` |
| `检查` | `ACTION` |
| `BMS` | `OBJECT` |
| `回复` | `ACTION` |
| `BEM` | `MESSAGE` |

### 8.2 参数标注

当文本中同时出现参数名称和值时，应分别标注。

```text
将CHM报文周期设置为500ms
```

| 片段 | 标签 |
| --- | --- |
| `CHM` | `MESSAGE` |
| `周期` | `PARAM_NAME` |
| `500ms` | `PARAM_VALUE` |

```text
发送报文ID为0x123的CHM报文
```

| 片段 | 标签 |
| --- | --- |
| `报文ID` | `PARAM_NAME` |
| `0x123` | `PARAM_VALUE` |
| `CHM` | `MESSAGE` |

### 8.3 故障表达

故障表达应标注自然语言中的原始表达，而不是标准化后的故障类型。

| 原始片段 | `FAULT_EXPR` | 标准化 `fault_type` |
| --- | --- | --- |
| `错误周期` | `错误周期` | `报文周期错误` |
| `周期不对` | `周期不对` | `报文周期错误` |
| `ID错误` | `ID错误` | `报文ID错误` |
| `数据域非法` | `数据域非法` | `报文内容错误` |
| `S+断开` | `S+断开` | `S+断路` |
| `PE断针` | `PE断针` | `保护接地故障` |

### 8.4 刺激动作与预期结果

发送故障报文、断开信号线、设置边界值通常属于测试刺激，不应标为预期结果。

```text
发送错误周期CHM报文，检查BMS能否正确回复BEM报文
```

- `发送错误周期CHM报文` 是测试刺激。
- `回复BEM报文` 是预期结果。

```text
断开S+通信线后，车辆应停止充电并记录故障
```

- `断开S+通信线` 是测试刺激。
- `停止充电`、`记录故障` 是预期结果。

### 8.5 通信对象与报文实体

`BMS`、`EVCC`、`SECC`、`车辆`、`充电机` 是 `OBJECT`，不标为 `MESSAGE`。

```text
检查BMS是否回复BEM报文
```

| 片段 | 标签 |
| --- | --- |
| `BMS` | `OBJECT` |
| `回复` | `ACTION` |
| `BEM` | `MESSAGE` |

## 9. 数据集格式

训练数据建议使用 JSONL 格式，每行一个样本。

```json
{
  "id": "SAMPLE_0001",
  "text": "发送错误周期CHM报文，检查BMS能否正确回复BEM报文",
  "entities": [
    {"start": 0, "end": 2, "label": "ACTION", "text": "发送"},
    {"start": 2, "end": 6, "label": "FAULT_EXPR", "text": "错误周期"},
    {"start": 6, "end": 9, "label": "MESSAGE", "text": "CHM"},
    {"start": 14, "end": 17, "label": "OBJECT", "text": "BMS"},
    {"start": 21, "end": 23, "label": "ACTION", "text": "回复"},
    {"start": 23, "end": 26, "label": "MESSAGE", "text": "BEM"}
  ],
  "labels": {
    "standard_source": "GB/T 34658-2025",
    "scene_type": "DC",
    "system_class": "unknown",
    "test_layer": "application_layer",
    "condition_type": "fault",
    "test_type": "negative",
    "test_stage": "低压辅助上电及充电握手阶段",
    "fault_type": "报文周期错误",
    "action_intent": ["发送报文", "检查响应", "故障注入"]
  },
  "expected_semantics": {
    "message_types": ["CHM", "BEM"],
    "parameters": {},
    "expected_results": ["发送BEM报文"]
  }
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `id` | 样本编号 |
| `text` | 原始自然语言测试需求 |
| `entities` | 实体标注列表，`start` 包含、`end` 不包含 |
| `labels` | 句级分类标签 |
| `expected_semantics` | 与当前可执行语义流程对齐的期望语义 |

## 10. 初始标注覆盖范围

第一版种子数据集至少覆盖：

- 协议一致性：物理层、数据链路层、应用层测试。
- 系统类别：A 类、B 类、C 类、未指定类别。
- 直流握手：`CHM`、`BHM`、`BEM`。
- 直流辨识：`CRM`、`BRM`。
- 时间同步：`CTS`。
- 参数配置：`CML`、`BCP`。
- 准备就绪：`CRO`、`BRO`。
- 充电阶段：`BCL`、`BCS`、`BSM`、`BMT`、`BMV`、`BSP`、`CCS`。
- 停止阶段：`BST`、`CST`、`BSD`、`CSD`。
- 错误处理：`BEM`、`CEM`。
- 报文故障：内容错误、周期错误、ID 错误、长度错误、格式错误、顺序错误、超时。
- 直流互操作性：接口连接、锁止、低压辅助电源、绝缘检测、S+/S-、K5/K6、保护接地。
- 交流互操作性：CP、CC、CC2、占空比、检测点电压、控制导引边界。
- 安全故障：过压、欠压、过流、急停、车辆中止、充电机中止、充电与行驶互锁。

## 11. 与模型任务的对应关系

| 模型任务 | 使用标签 |
| --- | --- |
| GlobalPointer 实体抽取 | `entities` |
| 标准来源分类 | `standard_source` |
| 场景分类 | `scene_type` |
| 系统类别分类 | `system_class` |
| 测试层级分类 | `test_layer` |
| 工况分类 | `condition_type` |
| 测试类型分类 | `test_type` |
| 阶段分类 | `test_stage` |
| 故障分类 | `fault_type` |
| 动作意图多标签分类 | `action_intent` |

模型输出后，还需要经过知识库约束与规则补全：

```text
模型候选语义 -> 报文/阶段/发送方校验 -> 接口/信号/故障合法性校验
-> 参数默认值补全 -> 时序知识库匹配 -> 动作库展开
```

其中报文字段映射使用 `data/message_field_knowledge/dc_message_fields.json`。
该知识库按 GB/T 27930-2023 的直流通信报文组织，用于把自然语言中的
`FIELD_NAME`、`PARAM_NAME`、`PARAM_VALUE` 映射到具体报文字段，例如：
