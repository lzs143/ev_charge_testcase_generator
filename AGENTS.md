# AGENTS.md

## 项目定位

本项目是“新能源汽车充电测试用例自动生成原型系统”的硕士论文实验代码库。目标是将自然语言形式的交流/直流充电测试需求转换为结构化测试用例。

第一阶段仅实现规则版原型，不实现深度学习模型、Web 服务或图形界面。代码结构需要便于后续替换为 BERT-CRF 或其他实体识别模型。

## 核心流程

自然语言测试需求 -> 文本预处理 -> 关键信息抽取 -> 测试约束表达 -> 流程模板匹配 -> 结构化测试用例生成 -> 规则校验 -> JSON/Excel 输出。

## 代码规范

- 使用 Python 3.10+。
- 使用 `dataclass` 定义核心数据结构，后续可替换或扩展为 `pydantic`。
- 所有模块、函数和方法应添加类型注解。
- 注释使用中文，说明业务含义或关键设计原因。
- 不引入和论文实验无关的 Web 服务、界面框架或后台任务。
- 算法实现应保持模块边界清晰，便于替换：
  - `extractor.py` 负责实体和关键信息抽取；
  - `template_matcher.py` 负责模板匹配；
  - `rule_checker.py` 负责规则校验；
  - `pipeline.py` 负责编排流程。

## 测试命令

```bash
pytest
```

建议每完成或修改一个模块后运行一次测试。

## 完成标准

- 新增或修改的模块具备基本单元测试。
- `pytest` 全部通过。
- 数据结构、输入输出字段与 README 中描述保持一致。
- 不在规则版原型阶段引入深度学习训练、推理服务或复杂依赖。
